# PIVOT 2026-05：從 SEO 矩陣轉顧問式數位產品

## 觸發原因

2026-05-04 用戶與策略合夥人（Claude）覆盤後確認：
- 1 個月鋪出 5 條戰線，0 收入
- SEO 矩陣在 AI Overviews 時代驗證週期 6-12 個月，但跑道只剩 5 個月
- 已有的 70+ 文章 / Agent-Reach / Gumroad 產品定位錯誤：被當「SEO 流量資產」在做，應該被當「諮詢產品的信任錨」在用

## 新方向

主線產品：**「AI 自動化診斷 + 工具包」**
- 三檔定價 NT$15,000 / 35,000 / 80,000
- 銷售模式：1 次診斷會議 + 1 份報告 + 工具模板包 + 30 天答疑
- 交付物 = 報告 + 模板，不寫定製代碼
- 90 天目標：2-3 個客戶
- 首選垂直行業：跨境電商代運 + 貨代物流（員工 30-200 人）
- 平行備選：中小型律師 / 會計 / 財稅顧問事務所（5-30 人，Day 30 切換用）

完整 plan 見 `~/.claude/plans/keen-doodling-glacier.md`（v2）和 `~/Desktop/claude/side-hustle-pivot/CLAUDE.md`

## 5 個 Agent 角色調整

### seo-writer（停產）
- **停止生成新文章**
- 已發布的 70+ 文章保留（被動跑 SEO 長尾）
- 不再追新關鍵字、不再發新內容

### content-ops（縮量）
- **不發新社群帖**
- 維護現有頁面的可用性（斷鏈 / 圖片失效 / SSL）
- 把高曝光低 CTR 頁面的內文 CTA 換 1-2 個到 `/diagnose`（小改，不要大動既有 HTML 結構）

### researcher（轉向）
- **不再找新聯盟 / 新關鍵字**
- 改為：用 Agent-Reach 抓目標客戶（貨代 / 代運公司）的招聘 JD / 痛點貼 / 融資新聞
- 輸出 sourcing CSV，每日更新

### strategist（守護）
- 監督所有 agent 不偏離 PIVOT 方向
- 每週檢查：是否有 agent 偷偷開始寫新文章 / 推新產品？發現 → 立刻拉回來
- 收入跟蹤：從「點擊 / 曝光 / AdSense」改為「診斷包銷售數 / 漏斗轉化率」

### builder（轉交付）
- **不做新 Gumroad 產品**
- 改為：建 `/diagnose` 著陸頁 + 診斷報告 Notion 模板 + n8n 工作流模板包
- 第一個客戶來後 → 快速做該客戶的定製報告（80% 套模板）

## 數位產品 / 聯盟連結的新角色

不再是「獨立收入流」，而是**診斷包的子模組**：
- Gumroad agent-skills 包 → 診斷包 NT$15,000 客戶的贈品
- Gumroad claude-crons → 同上
- 聯盟連結（DigitalOcean / Hahow / Systeme.io）→ 診斷報告裡推薦工具時附上（一份報告可能帶 5-10 個聯盟點擊，單個報告 NT$15,000 + 後續聯盟可能 NT$3-5K）

## 驗證關卡

- Day 14：`/diagnose` 頁 + 報告模板就位
- Day 30：發出 30+ 觸達 + 3 對話
