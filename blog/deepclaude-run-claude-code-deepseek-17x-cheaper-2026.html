DeepClaude 是什麼？原理解說

DeepClaude 由 GitHub 用戶 aattaran 開發，是一個輕量級的 bash script + API proxy 組合，核心原理非常簡單：

原理一句話：攔截 Claude Code 發出的 API 請求，把目的地從 Anthropic API 換成 DeepSeek / OpenRouter / Fireworks AI，然後把回應格式化成 Claude 格式返回。Claude Code 本身完全不知道後端被換掉了。

因為 OpenAI Compatible API 格式的廣泛採用，DeepSeek 和 OpenRouter 都支援與 Anthropic API 格式相容的 wrapper，DeepClaude 就是利用這個相容層做到「無縫替換」。

你的工作流程、快捷鍵、CLAUDE.md 設定、.claude/ 目錄下的所有 Agent Skills——全部照常運作。唯一改變的是後端 API 和費用。

步驟 1：取得 API Key（選一）

推薦：OpenRouter（最多模型選擇）

# 1. 前往 openrouter.ai 註冊，進入 Keys 頁面建立 API key # 2. 複製 key（格式：sk-or-v1-xxxxxxxxxxxx） # 3. 設定環境變數（加入 ~/.bashrc 或 ~/.zshrc） export OPENROUTER_API_KEY="sk-or-v1-你的key" source ~/.zshrc

備選：直接使用 DeepSeek API

# 前往 platform.deepseek.com 申請帳號（中國大陸服務，需VPN或用OpenRouter中轉） export DEEPSEEK_API_KEY="sk-xxxxxxxxxxxx"

步驟 2：安裝 DeepClaude

# 方法一：直接 curl 安裝（最快） curl -fsSL https://raw.githubusercontent.com/aattaran/deepclaude/main/install.sh | bash # 方法二：clone repo 手動安裝 git clone https://github.com/aattaran/deepclaude.git cd deepclaude chmod +x deepclaude.sh ./deepclaude.sh --setup # 設定完成後，加入 PATH（加入 ~/.zshrc） export PATH="$HOME/.local/bin:$PATH" source ~/.zshrc

步驟 3：在 Claude Code 中切換後端

# 啟動 Claude Code（正常方式） claude # 在 Claude Code 內執行指令切換模型 /deepseek ✓ 後端已切換至 deepseek/deepseek-chat-v3（OpenRouter） 費用：$0.14/M input，$0.28/M output # 切換其他模型 /model meta-llama/llama-4-scout ✓ 後端已切換至 meta-llama/llama-4-scout # 切回原始 Claude（需要時） /claude ✓ 後端已切換至 Anthropic Claude Sonnet 4.5 # 查看當前費用統計 /cost 本 session 費用：$0.023（DeepSeek V3，相當於 Claude 費用的 1/17）

進階設定：自動預設使用 DeepSeek

# 在 ~/.claude/CLAUDE.md 加入以下內容，讓每次啟動 Claude Code 自動使用 DeepSeek # （這樣就不需要每次手動切換） # 或設定環境變數強制使用 export DEEPCLAUDE_DEFAULT_MODEL="deepseek/deepseek-chat-v3" export DEEPCLAUDE_FALLBACK_TO_CLAUDE=true # 複雜任務自動切回 Claude
💡 建議策略：智慧混用
設定 DeepSeek 為預設，對於需要 Claude 原生能力的任務（如複雜的多步驟 agentic 任務、精確的函式庫 API 知識），用 /claude 臨時切回。一般的代碼生成、解釋、重構任務，DeepSeek V3 表現與 Claude Sonnet 相差不大。

常見問題：DeepClaude 值得用嗎？

DeepClaude 穩定嗎？會不會突然掛掉？
DeepClaude 本身只是一個 bash script + local proxy，不依賴任何外部服務器。它的穩定性取決於你選擇的後端 API（OpenRouter、DeepSeek）。OpenRouter 有 99.9% uptime SLA，一般使用非常穩定。設定 DEEPCLAUDE_FALLBACK_TO_CLAUDE=true 後，後端 API 如果有問題會自動切回原版 Claude，不影響你的工作流程。
DeepSeek 的程式碼生成品質真的夠嗎？
根據 2026 年 Q1 的多個 benchmark（HumanEval、MBPP、SWE-bench），DeepSeek V3 在程式碼生成上排名 Top 5，與 Claude Sonnet 4.5 差距在 5-10% 以內。一般的 CRUD 功能、API 整合、測試生成、重構任務，DeepSeek V3 表現與 Claude 相當。需要切回 Claude 的情況：涉及最新函式庫 API（訓練資料截止前的），以及需要複雜多步驟推理的 agentic 任務。
在台灣用 DeepSeek 有法律問題嗎？
台灣目前沒有禁止個人使用 DeepSeek API 的法規。但如果你的程式碼涉及政府標案、特定產業的個資法規範（如醫療、金融），需要確認相關合規要求。企業用戶建議諮詢法律顧問。個人開發者的 side project 和一般商業軟體開發，目前沒有已知法律障礙。
DeepClaude 跟 Claude Max 訂閱衝突嗎？
不衝突。Claude Max 訂閱包含的是 claude.ai 網頁版和 Claude Code 的使用量配額。DeepClaude 是在 API 層面替換，使用的是 OpenRouter/DeepSeek 的 API，不影響你的 Claude Max 訂閱。你可以同時保有 Claude Max（用於複雜任務）+ DeepClaude（用於大量日常任務）的混合方案，把 Max 配額留給真正需要的時候。
macOS / Linux / Windows 都支援嗎？
DeepClaude 的 bash script 原生支援 macOS 和 Linux。Windows 用戶需要使用 WSL2（Windows Subsystem for Linux）來運行，安裝步驟相同。DeepClaude 的 GitHub 上有 WSL2 的額外設定說明。
有沒有 GUI 版本？不太會用 terminal 可以嗎？
目前 DeepClaude 是 CLI 工具，沒有 GUI。但安裝過程只有 3 步驟，基本的 terminal 操作（貼上指令、按 Enter）就可以完成。社群有人在開發 GUI wrapper，可以關注 GitHub 的 Issues 追蹤進度。
