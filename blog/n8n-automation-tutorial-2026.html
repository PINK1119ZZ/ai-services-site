n8n 是什麼？為什麼台灣企業開始用它

n8n（念作「n-eight-n」，全名 node eight node）是一套開源的工作流程自動化平台。說白了就是：你可以把不同服務（像 Gmail、LINE、Google Sheets、Slack、ChatGPT 等）串在一起，設定好觸發條件，讓流程自動跑。

它的介面長得很像流程圖，你用「節點（Node）」代表每個步驟，用「連線」把步驟串起來。一個流程叫做一個「Workflow」。舉例：

  • 當有新訂單進來（觸發）→ 自動把資料存到 Google Sheets → 發 LINE 通知給業務
  • 每天早上 9 點（觸發）→ 抓昨天的銷售數字 → 自動彙整 → 發報表到群組
  • 有人填了 Google 表單（觸發）→ 發確認信給對方 → 在 Notion 建立一筆記錄

這類事情放在以前，你要嘛每天手動做，要嘛花錢請工程師寫程式。有了 n8n，很多流程可以自己拼出來，甚至不用寫一行程式碼。

為什麼台灣開始流行 n8n？

2024 年之前，台灣比較多人用 Zapier 或 Make（前身 Integromat）。n8n 是近兩年才在台灣的新創圈、電商圈、行銷圈快速擴散。主要原因有幾個：

  • 免費無限執行：自架版完全不限執行次數，Zapier 跑多了費用嚇死人
  • 可以寫程式擴充：有 JavaScript/Python Code 節點，做不到的事可以自己補邏輯
  • 資料不出境：自己架就自己管，對資安敏感的公司很重要
  • 400+ 內建整合：常用服務幾乎都有，沒有的可以用 HTTP Request 自訂
  • AI 節點崛起：2025 年 n8n 強化了 LangChain / OpenAI 的整合，做 AI 自動化流程超方便

n8n vs Zapier vs Make：怎麼選？

市面上三個主流的自動化工具各有適合的族群：

功能 n8n（自架） Zapier Make
費用 幾乎免費（VPS 約 NT$200/月） 免費版限 100 次/月，付費 USD$20/月起 免費版限 1,000 操作/月，付費 USD$9/月起
執行次數 無限 按 Task 計費 按 Operation 計費
介面 流程圖式，稍有學習曲線 最簡單，新手友善 流程圖式，中等難度
程式碼擴充 支援 JS/Python 基本不支援 有限度支援
自架 完全開源可自架 僅雲端 僅雲端
AI 整合 原生支援 OpenAI/LangChain 有但功能有限 有但需自行設定
適合對象 願意花時間設定、用量大、重資安 完全不懂技術、用量小 介於兩者之間

簡單判斷：如果你每月自動化執行次數超過 1,000 次，或是想做 AI 相關的流程，n8n 幾乎是最划算的選擇。如果只是偶爾跑幾個簡單通知，Zapier 免費版就夠了。

n8n 怎麼架設？3 種方法

不管哪種方法，核心都是：你需要一個地方「跑」n8n 的程式。

方法一：n8n Cloud（最簡單，但要付費）

直接到 n8n.io註冊，選方案，馬上可以用。不需要管伺服器，但每月 €20 起跳（約 NT$700），而且執行次數有上限。適合想快速試玩、不想碰伺服器的人。

方法二：Docker + VPS（推薦，性價比最高）

這是台灣工程師最常用的方法。你需要：

  1. 一台 VPS（DigitalOcean最低方案 USD$6/月，或 Vultr USD$6/月）
  2. 在伺服器上安裝 Docker
  3. 跑一行指令：docker run -d --name n8n -p 5678:5678 -v n8n_data:/home/node/.n8n n8nio/n8n
  4. 設定 Nginx + SSL（讓外部可以透過 HTTPS 存取，Webhook 才能接收）

設定完大概要花 1-2 小時，但設好之後就是免費無限用。如果你完全沒有伺服器經驗，可以找 AutoDev AI 幫你建好交給你，免費諮詢了解看看。

方法三：Zeabur / Railway（雲端快速部署）

台灣很流行的 PaaS 平台 Zeabur和 Railway 都支援一鍵部署 n8n，不需要自己管 Docker。費用依使用量計算，輕度使用可能比 VPS 便宜。適合對伺服器陌生但想嘗試的人。

5 個台灣中小企業最實用的 n8n 自動化案例

光看介紹可能感覺很抽象，直接看案例最清楚。以下 5 個都是真實場景，從最簡單到進階排列。

案例一：電商訂單自動通知（難度 ⭐）

場景：蝦皮、91APP 店家

每次有新訂單進來，自動把訂單資訊存到 Google Sheets，並發 LINE 推播通知給老闆和出貨人員，不用再一直盯著後台。

流程邏輯：

  1. Trigger（觸發）：Webhook 節點 — 蝦皮 / 91APP 有新訂單時送資料來
  2. Google Sheets 節點：把訂單號、品項、金額、收件人寫入 Sheets
  3. HTTP Request 節點：呼叫 LINE Messaging API，發推播給指定 LINE 帳號

省下的時間：每天手動查訂單 + 抄 Sheets 約 30-60 分鐘。

延伸閱讀：想了解 LINE 推播的更多應用，可以看 LINE 官方帳號自動回覆設定完整教學。

案例二：Google 表單自動報名流程（難度 ⭐⭐）

場景：課程、活動、診所報名

學員填完報名表單，系統自動發確認信、在 Notion 或 Sheets 建立資料、在行事曆新增活動提醒，老師 / 主辦人只要看 Notion 就能掌握所有名單。

流程邏輯：

  1. Google Sheets Trigger：偵測到試算表新增一列（表單送出）
  2. Gmail 節點：寄確認信給報名者，信件內容帶入對方填的姓名、課程名稱
  3. Notion 節點：在 Notion 資料庫新增一筆紀錄
  4. Google Calendar 節點（選配）：在行事曆新增活動

省下的時間：每次報名手動確認 + 整理名單約 10 分鐘；一個活動有 50 人報名就省了 8+ 小時。

案例三：競品價格監控自動通知（難度 ⭐⭐⭐）

場景：電商、零售業者

每天早上自動爬取競品網站的特定商品價格，和昨天比較，如果降價超過 5%，就發 Slack 或 LINE 通知給行銷團隊，立即反應定價策略。

流程邏輯：

  1. Schedule Trigger：每天早上 8 點觸發
  2. HTTP Request 節點：GET 競品網頁 HTML
  3. HTML Extract 節點：解析出目標商品價格
  4. Code 節點（JS）：比較今日 vs 昨日價格，計算降幅
  5. IF 節點：如果降幅 > 5%
  6. Slack / LINE 節點：發通知
  7. Google Sheets：記錄每日價格歷史

這個流程需要一點 JavaScript 知識處理資料比較，但整體框架還是拖拉完成的。

案例四：AI 客服自動分類回覆（難度 ⭐⭐⭐⭐）

場景：有大量客詢的電商 / 服務業

用戶透過 LINE 或 Email 傳來客服詢問，n8n 自動把問題送給 ChatGPT 判斷類型（退換貨 / 出貨查詢 / 產品問題），給出初步回覆，同時把問題分類記錄到 Sheets，人工客服只需要處理 AI 無法解答的複雜問題。

流程邏輯：

  1. Webhook Trigger：收到 LINE Webhook 事件
  2. OpenAI Chat Model 節點：把用戶問題送給 GPT-4o，附上 System Prompt（品牌資訊、常見問題知識庫）
  3. AI Agent 節點：讓 AI 決定要用哪個工具（查訂單 / 查庫存 / 直接回覆）
  4. HTTP Request 節點：呼叫 LINE API 回覆用戶
  5. Google Sheets：記錄問題類型、AI 回覆、是否需要人工接手

這個是進階案例，需要對 LINE Messaging API 有基本了解。可以搭配看 AI 客服完整指南了解整體架構。

案例五：每週自動產生營運報表（難度 ⭐⭐）

場景：小型電商、餐廳、工作室老闆

每週一早上自動彙整上週的銷售、新客數、客單價，用 AI 幫你寫一段重點摘要，以 Email 形式寄給老闆，省去手動整理報表的時間。

流程邏輯：

  1. Schedule Trigger：每週一 08:00
  2. Google Sheets 節點：抓取上週銷售資料（按日期篩選）
  3. Code 節點：計算總銷售額、新客數、平均客單價
  4. OpenAI 節點：把數字送給 GPT，請它用繁體中文寫一段簡短分析
  5. Gmail 節點：把報表和 AI 分析寄給指定 Email

這個流程不需要複雜的程式知識，全程用拖拉 + 少量設定就能完成，每週省下 1-2 小時整理報表的時間。

新手入門建議：從哪裡開始？

很多人看了 n8n 之後覺得很有趣，但不知道從哪裡下手。這邊給幾個實際建議：

Step 1：先想清楚你最痛的重複工作是什麼

不要為了用自動化而自動化。先列出你每天做 3 次以上的重複動作，圈出最花時間、最無聊的那個，那就是你的第一個自動化目標。

Step 2：從最簡單的「通知型」流程開始

通知型流程是入門首選：「某件事發生 → 傳一則訊息」。邏輯簡單、節點少、出錯機率低。等你熟悉了 n8n 的操作邏輯，再挑戰多步驟、有條件判斷的複雜流程。

推薦第一個流程

每天早上 9 點，自動把今天的 Google Calendar 行程整理成一條訊息，發到自己的 LINE。這個流程只需要 3 個節點，30 分鐘就能設好。

Step 3：善用 n8n 內建的 Workflow 模板

n8n 官方有個 Workflow Templates 資料庫，幾百個現成模板直接匯入就能用。你不用從零開始畫流程，找一個最接近你需求的模板，修改一下參數就好。

Step 4：懂一點點 JavaScript 讓你飛起來

n8n 的 Code 節點支援 JavaScript，讓你可以做自訂的資料處理。不需要很深，只要會：

  • 取某個欄位的值：$input.item.json.fieldName
  • 基本的字串處理和數字計算
  • 簡單的 if/else 判斷

用 ChatGPT 幫你寫這些小段程式碼完全沒問題，告訴它「我有一個 n8n Code 節點，輸入是 XXX，我要做 YYY，幫我寫 JS」，幾秒就有答案。

Step 5：善用社群資源

台灣已經有不少人在用 n8n，幾個資源值得收藏：

  • n8n 官方 Community Forum：遇到問題直接搜，八成都有人問過
  • Facebook 群組「n8n 台灣用戶」：有繁中討論，分享流程、問問題都可以
  • YouTube 搜 "n8n tutorial"：英文資源豐富，配合字幕學習效果很好
  • n8n.io/workflows：官方模板庫，依使用情境分類

n8n 的限制和注意事項

n8n 很強，但不是萬能的，有幾件事要提前知道：

  • 自架需要維護：伺服器更新、n8n 版本更新、偶爾的 Bug 排查都需要時間。如果你完全不懂伺服器，建議找人幫你設定好，或直接用 n8n Cloud。
  • 複雜邏輯還是需要程式底子：前面說了基礎流程不用寫程式，但要做真正複雜的判斷和資料處理，沒有程式背景很難進階。
  • 某些服務的 API 需要付費：n8n 本身免費，但你串接的服務可能要付費。例如 OpenAI API 依用量計費、某些 SaaS 的 API 要付費方案才能用。
  • Webhook 需要公開 HTTPS URL：如果要接收 LINE、蝦皮等外部的 Webhook，n8n 必須有可以從外部存取的 HTTPS 網址。本機測試可以用 ngrok 暫時解決，但正式環境一定需要有公開 IP 的伺服器。

如果你不確定 n8n 適不適合你的需求，或是架設過程遇到問題，歡迎透過 免費諮詢跟我們聊聊，我們幫台灣不少中小企業規劃過自動化流程。

常見問題（FAQ）

n8n 是免費的嗎？

n8n 是開源軟體，可以免費自行架設（Self-hosted）。官方也有雲端付費版（n8n Cloud），起價約每月 €20 歐元，包含 2,500 次工作流程執行。大多數小型團隊自己架在 VPS 上，每月只需負擔 VPS 費用（約 NT$200-500），幾乎零成本。

n8n 和 Zapier 有什麼差別？

最大差別是費用和彈性。Zapier 按執行次數收費，用量一大費用很高；n8n 自架版免費無限執行。n8n 還支援自訂 JavaScript/Python 程式碼節點，做不到的事情可以寫程式補，比 Zapier 彈性很多。但 n8n 初始設定門檻稍高一些，需要花時間了解介面和概念。

n8n 需要會寫程式嗎？

基礎的拖拉流程不需要寫程式。n8n 的視覺化介面讓你用連線的方式串接不同服務。但如果要做比較複雜的邏輯判斷或資料處理，懂一點 JavaScript 會讓你更如魚得水。完全不懂程式的人可以從簡單的通知型流程開始，再搭配 ChatGPT 幫你生成 Code 節點裡的程式碼。

n8n 可以串接 LINE Bot 嗎？

可以。n8n 有內建的 HTTP Request 節點，可以直接呼叫 LINE Messaging API。你可以做到：接收用戶訊息後觸發工作流程、自動發送 LINE 推播通知、把 LINE 訊息資料存到 Google Sheets 等。進階的 AI 回覆功能也可以透過 OpenAI 節點實現。

n8n 怎麼架設？需要伺服器嗎？

需要一台有公開 IP 的伺服器（或使用 n8n Cloud）。最簡單的方式是用 Docker 在 VPS 上部署，一台月費 NT$200-500 的入門 VPS 就夠用。DigitalOcean、Vultr、Linode 都是常見選擇。也有人用 Zeabur 或 Railway 快速部署。如果完全沒有伺服器經驗，也可以請 AutoDev AI 協助架設。

n8n 支援哪些服務串接？

超過 400 種服務，包含：Google Workspace（Gmail、Sheets、Drive、Calendar）、Slack、Discord、Telegram、LINE、Notion、Airtable、GitHub、Stripe、OpenAI/ChatGPT、HubSpot、Shopify 等。沒有內建的服務可以用 HTTP Request 節點自訂串接，幾乎任何有 API 的服務都能接。
