OpenAI Chronicle 是什麼？

2026 年 4 月，OpenAI 悄悄在 Codex for Mac 的研究預覽版中推出了一個讓人又愛又怕的新功能：Chronicle。

簡單說，Chronicle 就是「AI 幫你看螢幕並自動做筆記」。它會在背景持續截取你的螢幕畫面，透過視覺 AI 解析你在做什麼，然後把這些資訊轉化成結構化的 Markdown 記憶檔案，存在你電腦的本地目錄裡。

下次你跟 Codex 對話時，它就能說：「啊，你上週在 VS Code 裡處理那個 n8n webhook 的 bug，要繼續嗎？」——不需要你重新解釋背景。

這個功能一上線，立刻在開發者社群引爆兩種極端反應：

  • 🙌 「終於！這才是真正的 AI 助理，能記住我的工作脈絡！」
  • 😱 「等等，OpenAI 在截我的螢幕？我的密碼、客戶資料怎麼辦？」

這篇評測，我們從功能、教學到隱私爭議，幫你全面拆解 Chronicle，讓你自己決定——到底要不要開。

Chronicle 的核心機制：螢幕截圖 → AI 解析 → 本地記憶

在討論隱私之前，先搞懂它怎麼運作：

① 背景截圖（每隔數秒/分鐘）

Chronicle 以可設定的頻率截取你的螢幕畫面。預設不截取密碼輸入欄位和系統對話框，但這個過濾機制並不完美。

② 短暫上傳 OpenAI 伺服器

截圖會被傳送到 OpenAI 的視覺 AI 進行解析（辨識文字、代碼、視窗內容），完成後刪除。這個「短暫」是幾秒到幾十秒，目前 OpenAI 沒有提供精確時間。

③ 本地生成 .md 記憶檔

解析結果以結構化 Markdown 格式儲存在你電腦的 ~/.chronicle/memories/ 目錄，完全在你的機器上，OpenAI 不保留副本（根據官方說法）。

④ Codex 對話時調用記憶

當你和 Codex 對話時，它會自動讀取相關的 .md 記憶片段，像是一個「能記住你工作脈絡的同事」。

Chronicle 教學：如何開啟、查看與關閉記憶

開啟 Chronicle

Chronicle 目前是 Codex for Mac 研究預覽版的功能，需要 Mac Pro 或更高版本訂閱。如果你有資格使用：

  1. 打開 Codex for Mac 應用程式
  2. 進入 設定（Settings）→ Chronicle
  3. 開啟 Enable Chronicle 開關
  4. 選擇截圖頻率（建議先設為「低頻」熟悉後再調整）
  5. 設定排除應用程式清單（例如：銀行、密碼管理器）

查看和管理記憶檔案

Chronicle 生成的記憶檔案存在：

  • ~/.chronicle/memories/ — 按日期分類的 .md 記憶文件
  • ~/.chronicle/index.json — 記憶索引，Codex 透過這個查找相關記憶

你可以直接用 Finder 或終端機查看、編輯、甚至刪除這些文件。這些文件是純文字，沒有加密，所以如果你的電腦被入侵，這些記憶也會外洩。

暫停 Chronicle（最重要的操作）

當你需要處理以下內容時，務必先暫停 Chronicle：

  • 客戶合約、財務報表、個人資料
  • 密碼、API 金鑰、私密金鑰
  • 公司機密文件、NDA 相關資料
  • 任何不希望 AI 看到的畫面

暫停方式：Codex 選單列 → Chronicle 圖示 → 暫停（Pause）。或是使用快捷鍵（可自訂）。

🔒 使用 Chronicle？先保護你的 AI 傳輸安全

Chronicle 截圖上傳至 OpenAI 伺服器的過程，會透過你的網路連線傳輸。加一層 VPN 加密，讓傳輸過程不被攔截——台灣開發者最常用的選擇。

Chronicle 的隱私爭議：深度分析

讓我們把爭議說清楚，不要模糊帶過。

爭議核心：截圖「短暫上傳」到底是什麼意思？

OpenAI 官方說法是：截圖會被傳送到伺服器「僅用於 AI 視覺解析」，解析完成後「立即刪除」，OpenAI 不會用這些截圖訓練模型（需要你在設定中選擇退出訓練）。

⚠️ 你應該知道的疑慮

  • 「立即刪除」沒有獨立驗證機制——你只能相信 OpenAI 的說法
  • 如果 OpenAI 伺服器在這段傳輸窗口期間遭到入侵，截圖可能外洩
  • 截圖過濾機制（不截密碼欄）依賴系統 API，第三方應用的敏感欄位未必受保護
  • 多國隱私法規（如台灣《個人資料保護法》、歐盟 GDPR）對於「截圖含他人個資上傳美國伺服器」有潛在灰色地帶

✅ Chronicle 的資安保障措施

  • 記憶文件本地儲存，OpenAI 不保留最終記憶內容
  • 可完全關閉或暫停功能
  • 排除清單功能（特定應用不截圖）
  • 可選擇退出 AI 訓練資料（需手動設定）
  • 所有記憶文件可被用戶完整刪除

Microsoft Recall 前車之鑑

熟悉 AI 動態的讀者可能立刻想到：這不就是 Microsoft 的 Recall 功能嗎？

沒錯，Recall 和 Chronicle 概念非常相似。Microsoft Recall 在 2024 年因嚴重的隱私爭議（所有截圖本地儲存，但研究者發現可被惡意軟體輕易存取）被迫延遲推出，後來加入了 Windows Hello 生物認證保護和更嚴格的本地加密才重新上線。

Chronicle 和 Recall 的最大差異是：Chronicle 把截圖上傳伺服器處理，而 Recall 全程在本地處理（更隱私，但需要 NPU 算力）。兩者各有取捨。

如何用 Chronicle 又保護隱私：5 個立即可執行的方法

  1. 設定精確的排除應用清單
    在 Chronicle 設定中加入：銀行軟體、密碼管理器（1Password、Bitwarden）、公司 VPN 客戶端、電子郵件客戶端。這些應用視窗開啟時，Chronicle 自動不截圖。
  2. 使用 VPN 加密你的網路傳輸
    截圖上傳到 OpenAI 伺服器的過程透過你的網路進行。在公共 Wi-Fi 或未加密的網路環境下，VPN 能防止傳輸被攔截。
  3. 降低截圖頻率
    把截圖頻率設為「低」（每幾分鐘一次而非每秒），減少截圖總量和上傳次數，在效率和隱私之間取得平衡。
  4. 定期清空記憶目錄
    執行 rm -rf ~/.chronicle/memories/* 清空本地記憶，或只保留最近 7 天。舊的記憶如果落入他人之手，等同把你的工作歷史拱手相讓。
  5. 為記憶目錄加密
    使用 macOS 的 FileVault 加密整個磁碟，確保即使電腦遺失，記憶檔案也無法被存取。

Chronicle 2026 展望：OpenAI 的大局

Chronicle 本質上是 OpenAI 在「持久性 AI 記憶」領域的大賭注。

如果這個功能成熟落地，它代表的是：你的 AI 助理不再是每次對話從零開始的工具，而是真正「認識你」、「記得你做過什麼」的長期工作夥伴。這和 OpenAI 長期以來強調的 AI 不只是工具而是隊友 的願景一脈相承。

常見問題 FAQ

OpenAI Chronicle 是什麼？

Chronicle 是 OpenAI Codex for Mac 的研究預覽功能，它會持續截取螢幕畫面，自動將你的工作內容轉成本地儲存的 Markdown 記憶文件，讓 Codex 在後續對話中能記住你做過什麼。

Chronicle 的截圖會上傳到 OpenAI 伺服器嗎？

根據 OpenAI 說明，螢幕截圖會短暫上傳至 OpenAI 伺服器進行 AI 解析，之後刪除，最終記憶以 .md 文件形式儲存在本地。但「短暫上傳」這一點仍是主要爭議核心，尤其當你處理機密文件時。

Chronicle 和 Claude 記憶功能有什麼不同？

Claude 記憶功能需要手動設定，讓 AI 記住特定資訊。Chronicle 則是自動且被動的——它持續截圖並從螢幕影像生成記憶，不需要你主動描述。Chronicle 的覆蓋範圍更廣，但也因此引發更多隱私疑慮。

台灣用戶可以使用 Chronicle 嗎？

Chronicle 目前是 Codex for Mac 的研究預覽功能，需要 Mac Pro 或更高版本訂閱，且以美國/特定市場優先開放。台灣用戶可關注 OpenAI 官方公告，訂閱 Codex 後可能可以搶先測試。

使用 Chronicle 時如何保護個人隱私？

最直接的方式：(1) 處理機密文件時暫停 Chronicle；(2) 定期清空 ~/.chronicle/ 記憶目錄；(3) 使用 VPN 加密傳輸，降低截圖上傳時的資料外洩風險；(4) 在私人視窗處理敏感資料。
