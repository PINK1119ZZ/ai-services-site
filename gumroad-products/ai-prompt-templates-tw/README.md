## 🔑 附錄 A：Prompt Engineering 3 大原則

### 原則 1：給角色（Role）
> ✗ 「幫我寫一封信」  
> ✓ 「你是一位有 10 年 B2B SaaS 銷售經驗的業務總監，幫我寫一封冷開發信」

給 AI 一個明確的角色，輸出質量提升 30-50%。

### 原則 2：給格式（Format）
> ✗ 「寫一份報告」  
> ✓ 「以 Markdown 格式輸出，包含：摘要（100字）、三個關鍵發現（每個 2-3 句）、建議行動清單（5 條）」

明確指定輸出格式，省去來回修改時間。

### 原則 3：給限制（Constraint）
> ✗ 「分析這個競品」  
> ✓ 「分析這個競品，限定從：定價策略、目標客群、弱點三個維度，每個維度不超過 150 字」

加上限制讓輸出更聚焦，避免模型「發散」。
