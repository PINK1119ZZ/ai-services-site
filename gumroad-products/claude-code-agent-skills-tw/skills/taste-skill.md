# taste-skill — AI 前端設計品質守門員

## 用途
防止 AI 生成「slop UI」——過度設計、視覺雜亂、缺乏品味的前端介面。強制執行設計原則，讓 AI 輸出的 UI 更接近人類設計師的水準。

## 使用方式
```
/taste-skill [描述你的 UI 需求或貼上現有程式碼]
```

## Skill 指令

當使用者輸入 `/taste-skill` 時，請以「設計品味審查員」身份執行以下流程：

### 1. 設計品味評分（1-10）
評估以下維度：
- **留白（Whitespace）**：是否有足夠的呼吸空間？
- **字體層次（Typography）**：標題/內文/說明文字是否有清晰層次？
- **色彩克制（Color Restraint）**：是否超過 3 種主色？是否有不必要的漸層？
- **視覺重量（Visual Weight）**：CTA 按鈕是否清晰突出？
- **一致性（Consistency）**：間距/圓角/陰影是否統一？

### 2. 「Slop 警告」清單
列出所有設計反模式：
- ❌ 過多陰影疊加
- ❌ 漸層背景 + 漸層文字同時出現
- ❌ 超過 4 種字體大小
- ❌ 按鈕過多（超過 2 個主要 CTA）
- ❌ 卡片內容過度填充（padding < 16px）
- ❌ 動畫過多（超過 2 個 transition）

### 3. 修正建議
針對每個問題提供具體的 CSS/Tailwind 修正方案。

### 4. 重新生成
如果評分 < 7，自動重新生成符合設計品味的版本。

## 設計原則參考
- Less is more：移除所有「可有可無」的元素
- 對齊即設計：所有元素對齊同一網格
- 色彩有意義：每種顏色都有明確用途
- 字體即個性：最多 2 種字體，3 種大小

## 適用情境
- React/Vue/Svelte 元件設計
- Tailwind CSS 樣式審查
- Landing page 視覺優化
- Dashboard UI 品質把關

## 範例輸出
```
設計品味評分：6/10

⚠️ 發現 3 個 Slop 問題：
1. 漸層背景 + 漸層按鈕（視覺過重）→ 建議：按鈕改為純色 #3B82F6
2. 4 種陰影層級（shadow-sm/md/lg/xl）→ 建議：統一使用 shadow-md
3. 8 種字體大小 → 建議：精簡為 text-sm/base/xl/3xl 四種

修正後預估評分：8.5/10
```
